# 부동산 구독 카테고리 추가 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 설정 > 구독 카테고리에 "부동산 전체" 항목을 추가해, 사용자가 청약/대출/세금/재개발/전월세를 개별 구독하지 않고도 부동산 정책 알림 전체를 한 번에 받을 수 있게 한다.

**Architecture:** 파이프라인은 기사를 `category="부동산"`, `subcategory="청약"|"대출"|...`으로 이미 이중 분류하고 있다. Cloud Function의 구독 쿼리를 `array-contains-any [subcategory, category]`로 확장해 "부동산" 구독자가 모든 하위 기사를 수신하도록 하고, Android 앱에는 "부동산" 카테고리 항목과 해당 탭 필터 로직을 추가한다.

**Tech Stack:** Firebase Cloud Functions (Node.js 20), Android (Kotlin, Jetpack Compose, MockK, kotlinx-coroutines-test)

## Task 1: Cloud Function — 구독 쿼리 확장

**Files:**
- Modify: `functions/index.js:16-18`

현재 코드는 `policy.subcategory` 하나만 매칭한다. 이를 `[subcategory, category]` 로 확장해 "부동산" 구독자가 청약/대출/세금/재개발/전월세/fallback 기사를 모두 수신하게 한다.

- [ ] **Step 1: `functions/index.js` 수정**

`index.js` 17번째 줄 `const usersSnap = await db` 바로 위에 matchValues 선언을 추가하고, 쿼리를 교체한다.

```js
// 변경 전 (17번째 줄)
const usersSnap = await db
  .collection("users")
  .where("subscribed_categories", "array-contains", policy.subcategory)
  .get();

// 변경 후
const matchValues = [...new Set([policy.subcategory, policy.category])];
const usersSnap = await db
  .collection("users")
  .where("subscribed_categories", "array-contains-any", matchValues)
  .get();
```

`Set` dedup이 필요한 이유: fallback 기사는 `subcategory == category == "부동산"` 이므로 중복값이 생기고, Firestore `array-contains-any`는 중복값 입력 시 에러를 낸다.

- [ ] **Step 2: 수동 검증**

배포 전 로컬에서 구조가 올바른지 확인한다.

```bash

console.log(fallback); // ['부동산']
```
